COMMENCER

                API Endpoint

                https://klassrum.onrender.com/api/

l'API Klassrum fournit un accès dynamique et flexible aux données pour construire une plateforme éducationnelle robuste. Vous pourrez avoir la liste des enseignants, étudiants, sessions de cours programmées, etc.

Pour utiliser cet API, vous aurez besoin d'une clé API. Veuillez nous contacter à jackjosue517@gmail.com pour avoir votre clé API.

enseignants


# Voici un exemple avec curl
curl \
-X GET https://klassrum.onrender.com/api/teachers/

Pour avoir la liste de tous les enseignants disponibles, lancer une requête GET :
https://klassrum.onrender.com/api/teachers/

Pour ajouter un nouveau enseignant, utiliser une requête POST :
https://klassrum.onrender.com/api/teachers/


PARAMETRES DE LA REQUÊTE

Champ Type Description
firstname String Prénom(s) de l'enseignant
lastname String Nom de famille de l'enseignant
username String Nom utilisateur de l'enseignant
role String Statut de l'enseignant au sein de la structure
courses Array Unité(s) d'enseignement(s) dispensé(s) par l'enseignant
gender String (optionnel) - Genre de l'enseignant
address String (optionnel) - Adresse de l'enseignant
_id String (Non requis) Auto-genéré
registrationDate Date (Non requis) Auto-genéré

Pour supprimer un enseignant, lancer une requête DELETE :
https://klassrum.onrender.com/api/teachers/:id


PARAMETRES DE LA REQUÊTE

Champ Type Description
_id String Identifiant personnel de l'enseignant

Pour mettre à jour le statut d'un enseignant, utiliser une requête PUT :
https://klassrum.onrender.com/api/teachers/:id


PARAMETRES DE LA REQUÊTE

Champ Type Description
firstname String Prénom(s) de l'enseignant
lastname String Nom de famille de l'enseignant
role String Statut de l'enseignant au sein de la structure
courses Array Unité(s) d'enseignement(s) dispensé(s) par l'enseignant
gender String (optionnel) - Genre de l'enseignant

Succès

l'API de Klassrum utilise les codes de succès suivants:

Code de succès Signification
S01 La ressource spécifiée a été créée. JSON: { code: 'S01', description: 'RESSOURCE-CREATED' }
S02 La ressource spécifiée a été mise à jour. JSON: { code: 'S02', description: 'RESSOURCE-UPDATED' }
S03 La ressource spécifiée a été supprimée. JSON: { code: 'S03', description: 'RESSOURCE-DELETED', }
S04 L'authentification (connexion) a réussie. JSON: { code: 'S04', description: 'LOGGED-SUCCESS', ...}

Erreurs

l'API de Klassrum utilise les codes d'erreur suivants:

Code d'erreur Signification
E01 Impossible de récupérer les ressources données. JSON: { error, code: 'E01', description: 'UNEXCEPTED-GET-DATA' }
E02 Impossible d'envoyer les données vers la ressource spécifiée. Une erreur est survenue. JSON: { error, code: 'E02', description: 'UNEXCEPTED-ERROR-POST-DATA', }
E03 Une erreur est survenue lors de la mise à jour de la ressource. JSON: { error, code: 'E03', description: 'UNEXCEPTED-ERROR-UPDATE-DATA', }
E04 Impossible de supprimer la ressource indiquée. JSON: { error, code: 'E04', description: 'UNEXCEPTED-ERROR-DELETE-DATA', }
E05 Le serveur n'a pas pu crypter le mot de passe. Vérifier si le champ password est rempli. JSON: { error, code: 'E05', description: 'UNEXCEPTED-ERROR-HASH' }
E06 ATTENTION: À utiliser qu'en programmation!!! Veuillez ne pas indiquer l'erreur spécifique à l'utilisateur JSON: { code: 'E06', description: 'PASSWORD-NOT-MATCH' }
E07 ATTENTION: À utiliser qu'en programmation!!! Veuillez ne pas indiquer l'erreur spécifique à l'utilisateur JSON: { error, code: 'E07', description: 'UNEXCEPTED-ERROR-COMPARE', }
E08 La requête n'est pas authentifiée. Avez-vous envoyé l'en-tête Authorization ? Vérifier avec votre utilitaire de développement web JSON: { error, description: 'UNAUTHORIZED-REQUEST', code: 'E08' }
E09 ATTENTION: À utiliser qu'en programmation!!! Veuillez ne pas indiquer l'erreur spécifique à l'utilisateur JSON: { code: 'E09', description: 'USER-NOT-FOUND', }
